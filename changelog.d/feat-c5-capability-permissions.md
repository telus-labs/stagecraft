- **C5 — Capability-required permissions.** All four adapter `capabilities.json` files now declare `enforces.shell` and `enforces.network` (true for claude-code/codex/gemini-cli; false for generic). Four stages that need shell access to do their work declare `requiredCapabilities: { shell: true }` in `stages.js` (pre-review, qa, verification-beyond-tests, deploy). `assertCapabilities()` in `orchestrator.js` is called at dispatch time for every workstream — headless or interactive — and throws a named `CapabilityError` if the routed host lacks a required capability; the error surfaces as an `ESCALATE` gate with a "host does not enforce shell" message. 31 new tests in `tests/capabilities.test.js`.
