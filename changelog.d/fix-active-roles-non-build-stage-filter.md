- **`inferActiveRoles` no longer zeroes out non-build stages when `active_roles` is set.** When a stage-01 gate carries `active_roles: ["backend", "platform", "qa"]`, `computeDispatchPlan` intersected that list with each stage's own roles. For non-build stages such as design (`roles: ["principal"]`) the intersection was empty; an empty array is truthy, so `roles` was set to `[]`, the stage dispatched zero workstreams, wrote no gate file, and the driver looped until `max-iterations` killed it. Fix: `inferActiveRoles` now returns `null` (no filter) when the intersection is empty, meaning `active_roles` doesn't apply to this stage's role set. One regression test added to `tests/workstream-suppression.test.js`.
